# OpenTmAgent原始需求描述文档
>基于项目管理和团队管理的高级多功能管理平台，可在无LLM或小型化本地LLM的条件下使用。

## 原始需求描述

面向企业中层管理者，同时管理20+人，划分为多个小组的场景。
提供一个同时管理人员和项目进度的平台软件，通过Telegram Robot API或者本地CLI风格命令行访问，命令行访问的样式参考OpenCode和ClaudeCode的样式。
该软件的主要功能分2块：团队管理和项目管理

### 团队管理
#### 团队和人员档案
- [ ] 能够新增、删除、修改人员，每个[人员]包含姓名、工号、邮件地址、[人员能力模型]、所属[小组]、职位、当前执行[任务]列表、[责任田]
- [ ] 能够新增、删除、修改人员能力模型的维度、定义等级含义（以数字代表从1到5）。对每个员工的人员能力模型修改仅[admin用户]有权限。
- [ ] 能够新增、删除、修改[小组]，每个小组包括组名、组长、组员列表、责任田列表和关键人物列表。关键人物列表分为类型和员工名称，其中类型可自定义增加、删除、修改，默认的类型包括PL、MDE。每个人物均要求从已建立档案的人员中选择。
- [ ] 能够根据每个小组中的人员的任务列表以及任务的起始终止时间，绘制每个小组当前的正在进行的和即将到期的任务列表，估算每个人的任务负载并按照从高到低排名。估算每个小组从今天往后3个月内的任务负载压力曲线。小组负载是所有小组人员负载之和。负载的单位是人月。
- [ ] 能够以graph view的形式美观地展现每个小组、成员、责任田的关联关系。点击每个成员可进一步查看该员工的姓名、工号、邮件地址、[人员能力模型]、所属[小组]、职位、当前执行[任务]列表、[责任田]。该页面非admin用户可查看。
- [ ] 能够记录团队当月的总负载、平均人员负载、最高负载人员和负载量。
- [ ] 能够以雷达图的方式美观地展现每个人员的人员能力模型，人员能力模型的修改仅admin用户有权限。能够以雷达图的方式展现每个小组的总能力模型和平均能力模型。
#### 技术架构档案
- [ ] 能够存储、修改[模块]和子模块的关联关系。在前端提供基于方块手动拖拽方式描述模块和子模块来修改后端存储数据的方式（支持新增、删除方块，支持修改方块的名称，支持把一个方块A拖入另一个方块B中代表A是B的子模块。支持一个方块包含多个方块），每个方块代表一个模块或子模块。结果可导出为markdown mermaid图表。
- [ ]  能够存储、修改[功能]和功能的关联关系。在前端提供基于方块手动拖拽方式描述功能和子功能来修改后端存储数据的方式（支持新增、删除方块，支持修改方块的名称，支持把一个方块A拖入另一个方块B中代表A是B的子功能。支持一个方块包含多个方块），每个方块代表一个功能或子功能。每个方块可点击打开一个子页面，其中可编辑本功能依赖的多个模块，模块之间数据的交互传递顺序可以用线条连接。结果可导出为markdown mermaid图表。
- [ ] 能够存储、修改[责任田]和[功能]的关联关系。结果可导出为markdown mermaid图表。
- [ ] 能够新增、删除、修改[责任田], 每个责任田包含所属小组，责任田田主，责任田后备田主，责任田本年度[任务]。结果可导出为markdown mermaid图表。


### 项目管理
- [ ] 能够新增、删除、修改版本，每个版本包含以下关键角色：项目经理、软件经理、测试经理，每个关键角色仅记录姓名。每个版本包含多个[迭代]，每个迭代包含多个[任务]。
- [ ] 能够新增、删除、修改迭代，每个迭代包含多个任务，每个任务包含起始时间、终止时间、人月。每个任务之间存在依赖关系、关联关系。支持新增、修改、删除任务之间的依赖关系和关联关系。支持检查任务冲突（如依赖的任务的终止时间晚于当前任务的终止时间）。每个迭代还包含迭代终止时间和迭代起始时间，迭代人月工作量统计。支持检查迭代和任务冲突（如迭代的终止时间早于任何一个任务的终止时间）。支持修改某个任务和迭代的关联关系。某个迭代的任务可以依赖更早的迭代。
- [ ] 能够新增、修改、删除任务，每个任务包含特性交付责任人、开发责任人列表、测试人员列表、软件设计文档归档URL、依赖的任务列表、关联的任务列表。（依赖的任务需要先完成，关联的任务需要和当前任务同时完成）。每个以上人员除了测试人员外均需要已建立档案人员，测试人员仅记录姓名。
- [ ] 能够以graph view的形式渲染展示一个/多个版本的1个/多个迭代的任务交付依赖、关联关系。支持以节点的方式逐级展开。能够找到一个/多个版本的1个/多个迭代的最长路径。能够找到一个/多个版本的1个/多个迭代的负载最高人员。
- [ ] 能够以甘特图的形式展示一个版本的一个迭代的任务交付时间顺序和依赖、关联关系。结果可导出为markdown mermaid图表。
- [ ] 能够记录每个人员任务达成情况，按照提前完成、准时完成（完成时间在终止时间之前）、略微超期（完成时间在终止时间之后的24小时内）、严重超期（完成时间在终止时间之后的24小时以外）记录某个版本、某个迭代的每个任务的相关人员的任务总数、达成个数、未达成个数、提前完成、准时完成、略微超期、严重超期个数。结果可导出为excel表格。



### 安装、部署、用户管理要求
- [ ] 支持提供脚本自己创建docker实例一键部署。
- [ ] 支持使用nginx作为代理服务器进行高并发访问
- [ ] 支持admin用户和非admin用户登陆，对于admin用户额外要求输入密码，密码仅提供首次设置，后续不再更新。非admin用户要求输入来访者工号，非admin用户不要求输入密码。
- [ ] 能够记录每个数据修改操作的操作日志，记录操作时间，操作结果和操作用户（admin或非admin用户的工号）
